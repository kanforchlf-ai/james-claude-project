## ⚠️ 自動生成 vs 手寫頁面

### 自動生成（**改了會被覆蓋，不要直接動**）

`update_dashboard.py` 每跑一次就**完全覆蓋**這些檔案：

```
index.html              ← 首頁
weekly.html             ← 本週點名
trend.html              ← 出席趨勢
cowork.html             ← 配搭出席總覽
kids/index.html         ← 兒童專區

y1, y2, y3/             各區 index.html、invite.html、leaderboard.html
hs1, hs2, hs3/          各區 index.html、invite.html、leaderboard.html
ms1, ms2/               各區 index.html、invite.html、leaderboard.html
```

要改這些頁面的版面/內容 → **去 `update_dashboard.py` 改產生 HTML 的程式碼**，不要改 HTML 本身。

### 手寫（**可以直接改**）

```
81Y3-dashboard/
├── youth.html              ← 青年群組頁
├── hs.html                 ← 高中群組頁
├── ms.html                 ← 國中群組頁
├── guide.html              ← 系統說明

├── cowork-checkin.js       ← 配搭打卡互動邏輯（連 Supabase）
└── cowork-checkin.css      ← 配搭打卡樣式
```

LINE Bot 相關也是手寫：`bot_server.py`、`dashboard_rag.py`、`Dockerfile`、`requirements.txt`、`LINE_BOT_README.md`。

⚠️ 注意 `cowork.html` 是**混合架構**：頁面骨架由 `update_dashboard.py` 生成（不可直接改 `cowork.html`），互動部分在手寫的 `cowork-checkin.js`（可直接改）。詳見下方「配搭打卡 / Supabase 後端」段落。

## 配搭打卡 / Supabase 後端

`cowork.html` 是混合架構：

- **頁面骨架**：`update_dashboard.py::update_cowork_page()` 自動生成，含 `#checkin-bar` / `#leaderboard` 注入點 + 卡片 `data-name` / `data-zone` 屬性
- **互動邏輯**：手寫 `cowork-checkin.js`（不會被覆蓋）
- **後端**：Supabase 表 `cowork_checkins`

### 想改 Tier 切點（例如改成「頂標前 5 名」）

`cowork-checkin.js` 裡找 `LEADERBOARD_TIERS`：

```javascript
const LEADERBOARD_TIERS = [
  { name:'medal',  range:[1, 3],   ... },
  { name:'top',    range:[4, 10],  label:'🌟 頂標 (4-10)',  ... },
  { name:'upper',  range:[11, 25], label:'⭐ 前標 (11-25)', ... },
  { name:'median', range:[26, 40], label:'✦ 中標 (26-40)',  folded:true, ... },
  { name:'rest',   range:[41, Infinity], folded:true, ... },
];
```

改 `range` 跟 `label` 即可 → B. 內層 push。`folded:true` 表示預設摺疊。

### 想改競賽時間（例如改成週六開賽）

`cowork-checkin.js` 開頭找：

```javascript
const COMP_START_HOUR = 11;   // 開賽小時（台灣時間）
const COMP_START_MIN = 30;
const COMP_DURATION_HOURS = 35.5;  // 35.5h = 週日 11:30 ~ 週一 23:00
```

`getCompetitionState()` 假設「開賽日 = 週日」(`day === 0`)。改成別的開賽日要連同改 `getCompetitionState` 裡的 `setUTCDate(date - day)` 邏輯（日期偏移）。

### Supabase 後端

| 項目 | 值 |
| --- | --- |
| Project URL | `https://hiytxefiylgsjehzxglw.supabase.co` |
| Dashboard | <https://supabase.com/dashboard/project/hiytxefiylgsjehzxglw> |
| 表 | `cowork_checkins` |
| 欄位 | `id`(uuid) / `week`(text, `YYYY-MM-DD`) / `zone`(text) / `member_name`(text) / `checked_at`(timestamptz) |
| Unique 約束 | `(week, member_name)` — 一週一人一筆 |
| RLS 政策 | anon 可 SELECT / INSERT / DELETE，**不開 UPDATE**（避免時間被竄改）|

**常用 SQL**：

```sql
-- 看本週打卡狀況
select * from cowork_checkins
 where week = '2026-05-03'
 order by checked_at;

-- 清某週測試資料
delete from cowork_checkins where week = '2026-05-03';

-- 看歷年戰績
select week, count(*) as 完成人數
  from cowork_checkins
 group by week
 order by week desc;
```

### 想加新功能（例如「歷年戰績頁」）

通常需要：
1. 確認 Supabase 表能滿足查詢（多週查詢 → 已支援）
2. 寫獨立頁面 `81Y3-dashboard/cowork-history.html`（歸入手寫類）
3. 從 `cowork-checkin.js` 抽 helper 出來共用，或寫獨立 JS
